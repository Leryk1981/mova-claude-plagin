# Testing Patterns

This skill provides guidance for writing and maintaining tests.

## Capabilities

- Unit testing with Jest, Vitest, Mocha
- Test-driven development (TDD) workflow
- Mocking and stubbing strategies
- Code coverage analysis

## When Active

This skill is automatically loaded when user prompts mention:
- test, tdd, mock, coverage, jest, vitest, spec

## Guidelines

1. Write tests before implementation (TDD)
2. Test behavior, not implementation details
3. Use descriptive test names
4. Keep tests isolated and independent
5. Aim for meaningful coverage, not 100%
